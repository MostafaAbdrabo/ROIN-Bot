# SECTION B: MONTHLY SCHEDULE PLAN SYSTEM
====================================================================

## B1. What is it?
At the start of each month, department managers create a plan showing:
- Which employees work which days
- Day shift vs Night shift
- Which of the 5 locations each employee works at
- Which days are planned leave/off/rest

## B2. Work Locations (5 total)
- Office
- Russian Kitchen
- Egyptian Kitchen
- Pastry-Soup Kitchen
- FoodTruck & Bakery

Add "Work_Location" column to Employee_DB (already added in sheet).
Each employee has a DEFAULT location, but can be assigned to different
locations on different days in the monthly plan.

## B3. Google Sheet Tab: Monthly_Schedule
Already created with these columns:
Plan_ID, Month, Department, Location, Emp_Code, Emp_Name(VLOOKUP),
Shift (Day/Night/Off/Leave/Rest),
D1 through D31 (values: W=Work, N=Night, OFF=Day Off, V=Vacation,
S=Sick, U=Unpaid, B=Business Trip, H=Holiday, R=Rest),
Uploaded_By, Status, Created_At

Status flow: Draft → HR_Review → Manager_Review → Approved → Active

## B4. Bot Flow — Creating a Monthly Plan

### Option 1: Upload Excel (preferred for large departments)
1. Manager taps 📅 Monthly Schedule → 📤 Upload Plan
2. Sends an .xlsx file with columns: Emp_Code, Shift, D1-D31
3. Bot reads the file, auto-fills Month + Department + Location from Employee_DB
4. Shows preview: "15 employees, March 2026, Egyptian Kitchen"
5. Manager confirms → saved to Monthly_Schedule tab as "Draft"
6. Notification available for HR to review

### Option 2: View/Edit Plan in Bot
1. Manager taps 📅 Monthly Schedule → 📋 View Current Plan
2. Bot shows list of employees in their department
3. Each employee shows: name + this week's schedule
4. Manager can tap an employee → change a specific day
5. Changes saved immediately

## B5. Review & Approval Flow
1. Manager creates/uploads plan → Status = "Draft"
2. HR_Manager or HR_Staff reviews → can edit → Status = "HR_Review"
3. Catering Director (or relevant Director) approves → Status = "Approved"
4. Once approved → Status = "Active" — this is the live plan

## B6. PDF Export — LANDSCAPE FORMAT
When plan is approved, generate PDF using the company Word template:
- LANDSCAPE orientation (already in JD_Template.docx)
- Company header (logo + address)
- Title: "[Department] — Monthly Schedule — [Month Year]"
- Table: Employee names down the left, days 1-31 across the top
- Each cell shows the status (W/N/OFF/V/S/H/R) with color coding
- Color legend at bottom
- Footer: "Generated by ROIN WORLD FZE HR System"
- Save to Drive: /Schedule_Plans/[YEAR]/[MONTH]/[Department].pdf

Use fpdf2 in LANDSCAPE mode. Match the template style.

## B7. Who Sees What

| Role | Can See | Can Edit |
|------|---------|----------|
| Bot_Manager | All departments, all locations | Everything |
| Director | All departments, all locations | Read only |
| Catering Director (Direct_Manager of kitchen) | Kitchen departments | Approve |
| HR_Manager | All departments | Review + edit |
| HR_Staff | All departments | Review |
| Direct_Manager | Own department only | Create + edit own |
| Supervisor | Own team only | View only |
| Employee | CANNOT see the plan | — |

# SECTION C: BOT FEEDBACK SYSTEM
====================================================================

## C1. What is it?
ANY employee can report bugs, send suggestions, or ask questions about the bot.

## C2. Two Categories
1. 🐛 Bug Report — something is broken or not working
2. 💡 Suggestion — idea to improve the bot

## C3. Bot Flow
1. ANY role → main menu has: "📬 Bot Feedback"
2. Select: 🐛 Bug Report / 💡 Suggestion / ❓ Question / 😤 Complaint
3. Type subject (short title)
4. Type description (detailed)
5. Optional: upload photos (1-3 screenshots)
6. Summary → Submit
7. Saved to Bot_Feedback tab
8. Bot_Manager sees all feedback in their menu

## C4. Bot_Manager Feedback Dashboard
Bot_Manager → 📬 Feedback Management:
- Open tickets: X | Resolved: X | This month: X
- View by type: Bugs / Suggestions / Questions / Complaints
- Tap ticket → see full detail + photos
- Respond → response saved + employee notified
- Change status: Open → In_Progress → Resolved → Closed

## C5. Google Sheet Tab: Bot_Feedback
Already created:
Ticket_ID, Date, Emp_Code, Emp_Name(VLOOKUP), Department(VLOOKUP),
Type, Subject, Description, Photos, Status, Response, Response_Date, Responded_By

## C6. Ticket IDs

- Suggestion: SUG-YYYY-NNNN
- Question: QST-YYYY-NNNN
- Complaint: CMP-YYYY-NNNN
